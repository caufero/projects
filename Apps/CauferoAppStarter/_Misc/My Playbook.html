Builder Guide: AI Agent Interaction
Your simple playbook for building apps with the CauferoAppStarter AI Agent. Control the build phase by phase. Request, approve, paste, verify, repeat. This guide defines exactly how you interact with the agent from requirements to deployment.
Builder Workflow Phase-Based Copy-Paste Prompts Verify & Approve
🎯 The Golden Rule One phase at a time

The Golden Rule

You ask for one phase at a time, then you implement, verify, and only then move on.

How This Works

  1. You control the build - Request each phase explicitly
  2. The agent builds in phases - Delivers one complete package at a time
  3. You paste and run - Implement the output in FileMaker
  4. You verify - Test the results before moving forward
  5. You approve - Explicitly signal readiness for the next phase

Why phase-based? Each phase builds on the previous. Skipping verification creates cascading errors. This method ensures quality and prevents rework.

📊 The Build Phases Phase 0 through Phase 5
0

Phase 0: Start

You paste requirements

You send:

  • The full client requirements
  • Any screenshots, forms, spreadsheets
  • Roles and permissions (if available)

Then you say:

Phase 1. Generate tables and fields.
1

Phase 1: Tables and Fields

Schema definition

You ask:

Phase 1: Based on these requirements and CauferoAppStarter rules, list every table and field needed.

You do:

  • Review schema output
  • Approve or correct it

You reply with either:

  • Approved. Phase 2.
  • Adjust schema: (your bullet changes)
2

Phase 2: Pages and Modals Inventory

Complete UI inventory

You ask:

Phase 2: List every page and modal needed. Group by module. For each page, include purpose, primary table, supporting tables, and actions.

You do:

  • Review the list
  • Approve or correct it

You reply with either:

  • Approved. Phase 3.
  • Adjust pages: (your bullet changes)
3

Phase 3: Links Plan and Link Creation

Navigation structure

You ask:

Phase 3: Based on the pages list, generate all links needed and the exact method to create them in CauferoAppStarter. If scripts apply, output the full script.

You do:

  • Paste the script (or follow manual steps) in FileMaker
  • Run it
  • Verify the links appear and route correctly

You reply:

Links created. Phase 4.
4

Phase 4: Page-by-Page Build Loop

Build one page at a time

You ask:

Phase 4: Generate the full script for page: [Page Name]. Include dependencies (value lists, custom functions, helper scripts, routing edits) if needed.

You do:

  • Create or update the script in FileMaker
  • Run it
  • Verify the page works

Then you ask the next one:

Next page: [Page Name].

Repeat until all pages are done.

5

Phase 5: Edits and Fixes

When something changes

When a script must be modified, you paste the current script and give bullet points.

You ask:

Phase 5: Modify this script using these points: (bullets). Output the full edited script steps. Also list side impacts.

You do:

  • Replace the script steps in FileMaker
  • Test again
✅ How You Approve Each Phase Keep approvals clean and concise

Use short approvals to keep the build clean:

Simple Approval
Approved. Continue.
Next Phase
Approved. Next phase.
Request Changes
Change these 3 things:
(then bullet points)
📝 What You Must Provide When Asking For a Page Required information for Phase 4

When you request a page, always include:

  • Page name (exact)
  • What it should show (columns or sections)
  • Key actions needed (add, edit, delete, view, search, filter)
  • Any special widgets (text editor, dropdown, charts, uploads)

Example Prompt

Phase 4: Generate the full script for page: Staff List. Show columns: Name, Role, Phone, Status. Actions: Add Staff, View Staff, Disable Staff. Search across Name and Phone. Filters: Status.
🔍 Verification Checklist Simple validation per phase

After Phase 1: Tables & Fields

  • ☑ Tables exist
  • ☑ Fields exist with correct types
  • ☑ Foreign keys exist and are indexed
  • ☑ Audit fields exist
  • ☑ Naming matches conventions

After Phase 2: Pages Inventory

  • ☑ Every workflow has a screen
  • ☑ Every entity has list + details
  • ☑ Child items have modals
  • ☑ Setup pages exist if needed

After Phase 3: Links

  • ☑ Links show in menu
  • ☑ Ordering is correct
  • ☑ Parent groups work
  • ☑ Clicking routes correctly

After Each Page (Phase 4)

  • ☑ Page loads without errors
  • ☑ SQL returns correct rows
  • ☑ Actions work
  • ☑ Modals open and close
  • ☑ Save works and validates

👥 Your Job vs The Agent's Job Clear division of responsibilities

Your Job

  • Provide requirements
  • Approve outputs phase by phase
  • Paste scripts into FileMaker
  • Run scripts
  • Test and report issues

Agent's Job

  • Read docs first
  • Follow CauferoAppStarter rules
  • Output correct phase packet only
  • Include dependencies and edits
  • Give you verification steps
