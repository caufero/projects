Populate Available Links Script Documentation
Navigation payload generator for CauferoAppStarter. Creates Link Records for sidebar menus, hamburger navigation, tiles, quick links, and module selectors. Uses strict delimiter contract (pipe-separated fields, paragraph-separated records) to ensure deterministic parsing by downstream UI renderers.
Navigation Link Records Menu Builder Permissions
📖 Overview Purpose and Integration

Purpose

The Populate Available Links script generates the app's navigation payload, called Link Records, used to build all navigation menus including sidebar, hamburger menu, tiles, quick links, and module selectors.

This script outputs a single text blob containing multiple link records. Each record describes either a Section Header (top-level group) or a Link Item (clickable screen under a section).

Where This Script Fits

  1. User logs in or app loads a module selector
  2. App determines user's role, permissions, or enabled modules
  3. Populate Available Links generates navigation payload based on permissions
  4. UI scripts render menus using the payload (often inside WebViewer HTML/JS)

Hard Rule: Keep the delimiter contract stable. Change values only, never structure.

📋 Output Data Contract Link Records Structure

Records are separated by paragraph mark ¶. Fields within each record are separated by pipe |. Each record has 5 required fields and up to 2 optional fields (ParentLinkID and Flags).

Position Field Name Required Description
1LinkID✓Unique ID (UUID recommended)
2OrderKey✓Sort and hierarchy code (e.g., 3, 3.1)
3Caption✓Display title in the menu
4Subtitle✓Secondary label (may be empty)
5IconPath✓SVG path string
6ParentLinkID—LinkID of the parent section (empty for sections)
7Flags—UI directives (e.g., "After Section Divider")

Home Overview Menu Prompt Script
About this script
Navigation payload generator
Generates Link Records for all navigation menus
Outputs pipe-separated, paragraph-delimited text
Permission-based filtering by user role
Supports hierarchical menu structure
Used by sidebar, hamburger, tiles, quick links
Critical: Maintains stable delimiter contract
Field Structure (7 fields)
Required and optional fields
1. LinkID - Unique identifier (UUID)
2. OrderKey - Sort/hierarchy (e.g., 3, 3.1)
3. Caption - Display title
4. Subtitle - Secondary label
5. IconPath - SVG path data
6. ParentLinkID - Section parent (optional)
7. Flags - UI directives (optional)
Delimiter Contract
Critical separators
Field separator: Pipe (|)
Record separator: Paragraph (¶)
Records end with ¶
Empty lines ignored by parsers
No spaces around pipes
Empty fields remain empty (not "null")
Record Types
Section vs Item
Section Header: ParentLinkID is empty
Link Item: ParentLinkID points to section
OrderKey must match parent hierarchy
Sections can have "After Section Divider" flag
Validation Checklist
Pre-exit checks
✓ Output not empty for permitted users
✓ All records have 5+ fields
✓ LinkIDs are unique
✓ ParentLinkIDs match existing sections
✓ OrderKey consistent with hierarchy
✓ No unescaped pipes in IconPath
✓ Payload ends with ¶
Common Mistakes
What to avoid
❌ Inconsistent OrderKey vs ParentLinkID
❌ Malformed or missing UUIDs
❌ Pipes inside IconPath
❌ Changing delimiter contract
❌ Adding/removing fields
❌ Extra spaces around separators
