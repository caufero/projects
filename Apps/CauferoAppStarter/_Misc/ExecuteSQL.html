ExecuteSQL Patterns
Home Prompt
About this prompt
Canonical ExecuteSQL generation for FileMaker
Paste this prompt into your AI agent along with your query goal
AI generates FileMaker scripts with strict formatting rules enforced
All table and field names quoted, keywords uppercase, clean spacing
Safe parameter binding with ? placeholders
Proper Let pattern and result parsing with loop initialization
Output is production-ready FileMaker script code
Hard Rules (Non-Negotiable)
6 mandatory formatting rules
1. Quote all table and field names with \"...\"
2. SQL keywords UPPERCASE (SELECT, FROM, WHERE, etc.)
3. Clean clause spacing (SELECT...FROM...WHERE...)
4. No extra spacing between GROUP BY and ORDER BY
5. Query must match the stated goal
6. Use CauferoAppStarter Let pattern
SQL Clauses
Main components
SELECT FROM LEFT JOIN WHERE GROUP BY ORDER BY HAVING COUNT DISTINCT
Canonical Flow
5-step pattern
1. Run ExecuteSQL() inside Let block
2. Save result into $List variable
3. Compute $Total = ValueCount($List)
4. If $Total > 0, loop and parse rows
5. Else, set defaults if needed
Separators Contract
Fixed delimiters
fieldseparator = "|" rowseparator = "¶"
Key Concepts
Important terms
QUOTED NAMES UPPERCASE KEYWORDS PARAMETER BINDING LET PATTERN $i = 0 GetValue() Substitute()
WHERE Condition Examples
Flexible, goal-driven patterns
Single ID: WHERE t1.\"ID\" = ?
Multiple: WHERE t1.\"Dept\" = ? AND t1.\"Status\" = ?
OR logic: WHERE t1.\"Status\" = ? OR t1.\"Status\" = ?
Date range: WHERE t1.\"Date\" >= ? AND t1.\"Date\" <= ?
Search: WHERE t1.\"Name\" LIKE ?
Special Field Handling
Required conversions
Description fields: ConvertLineBreakMarkersToReturns()
Dates: MySQLDateToFMPDateText() when needed
Keep MySQL date text for YYYY-MM-DD format
