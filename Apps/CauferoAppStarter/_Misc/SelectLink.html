Select Link Script Documentation
Central navigation router for CauferoAppStarter. Receives Link ID, queries Links table for destination metadata (Layout, Parent Link ID), applies UI cleanup (modal/banner/menu handling), then routes to correct page script based on Layout value. Data-driven navigation from anywhere in the app. Supports modal handling, menu synchronization, app-specific routing overrides.

About this script
Central navigation router
Data-driven navigation using Links table
Receives Link ID, queries destination metadata
Routes based on Links::Layout field value
Handles modal windows, banner refresh, menu sync
Navigate consistently from anywhere in the app
Maps route keys to page rendering scripts
High-Level Flow (8 steps)
Navigation sequence
1. Capture Link ID from script parameter
2. Query Links table for Layout and Parent Link ID
3. Handle Modal layout (close, refresh banner, exit)
4. Close menu window if not in Persistent Side Menu mode
5. Always refresh banner (+++ Banner Page)
6. Sync persistent side menu if applicable
7. Route to destination based on $$Layout Name
8. Fallback: clear $$Page, refresh window if no match
Global Variables
Set and read by script
$$Link ID - Selected Link record ID (set)
$$Layout Name - Route key from Links table (set)
$$Parent Link ID - Menu hierarchy (set)
$$Search Item - Cleared on navigation (set)
$$Startup - First-time startup detection (read)
$$Menu Mode - Menu behavior control (read)
$$App Name - App-specific routing (read)
$$Page - Cleared in fallback (set)
Links Table Structure
Required fields
Links::ID - Primary key, passed as parameter
Links::Layout - Route key (e.g., "Patients Database")
Links::Parent Link ID - Menu hierarchy grouping
SQL: SELECT "Layout", "Parent Link ID" FROM "Links" WHERE ID = ?
Field names must match exactly (quoted in SQL)
Routing Patterns (4 types)
Common route structures
One Key → One Page: Layout = "X" → +++ X Page
Multiple Keys → Same Page: Layout = "X" OR "Y" → +++ Page
Key + App Condition: App = "Bridge" AND Layout = "X"
Key + App Exception: Layout = "Dashboard" AND App ≠ "Bridge"
Critical Routing Rules
Must follow
Rule 1: Links::Layout must match route key exactly
Rule 2: "Layout" field is route key, not literal layout name
Rule 3: Banner refresh happens before routing (always)
Rule 4: Menu mode controls window close behavior
Case-sensitive matching, extra spaces break routing
Adding New Navigation (5 steps)
Extension procedure
Step A: Decide route key (e.g., "Suppliers Database")
Step B: Create destination script (+++ Suppliers List Page)
Step C: Add Else If branch to Select Link router
Step D: Create/update Links record with Layout value
Step E: Test in Modal, non-persistent, persistent menu modes
Troubleshooting
Common issues
Link does nothing: Check $$Link ID, Links::Layout value match
Always hits fallback: Check for typos, extra spaces in Layout
Menu window stays open: Check $$Startup, $$Menu Mode
Banner doesn't update: Check +++ Banner Page script
