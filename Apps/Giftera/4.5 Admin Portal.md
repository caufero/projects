  <h1 id="giftera-admin-portal-content-spec-final-copy-for-developers">Giftera Admin Portal Content Spec (Final Copy for Developers)<a class="h-anchor" href="#giftera-admin-portal-content-spec-final-copy-for-developers"></a></h1>
<p>Version: v1.0<br/>

<h2 id="1-introduction">1) Introduction<a class="h-anchor" href="#1-introduction"></a></h2>
<h3 id="admin-portal-purpose-short">Admin Portal Purpose (Short)<a class="h-anchor" href="#admin-portal-purpose-short"></a></h3>
<p>The admin portal is the internal control room for Giftera.<br/>
It is used to manage users, gifts, templates, payments, notifications, USSD activity, AI usage, and system configuration.<br/>
Every admin action must be traceable in Audit Logs.</p>
<h3 id="global-copy-rules">Global Copy Rules<a class="h-anchor" href="#global-copy-rules"></a></h3>
<ol>
<li>Use exact words. Avoid vague text like "maybe" or "try later" unless the system truly cannot act.</li>
<li>Always show what will happen after a button click.</li>
<li>Every list must have: search, filters, pagination, and export (where relevant).</li>
<li>Every destructive action must have a confirmation modal.</li>
<li>Every financial number must display currency: <strong>GHS</strong>.</li>
<li>Every timestamp must show timezone: <strong>GMT</strong>.</li>
<li>Every status must be plain: Pending, Delivered, Read, Failed, Queued, Sent, Delivered by Provider, Failed.</li>
</ol>
